## Use issues for

- bug reports
- documentation corrections
- product feedback
- feature requests
- onboarding friction reports

## Before opening an issue

- check the [FAQ]({{ site.baseurl }}/docs/faq/)
- check [Troubleshooting]({{ site.baseurl }}/docs/support/troubleshooting/)
- check [Availability notes]({{ site.baseurl }}/docs/support/availability/) if the question is about continuity or web caveats

## Open an issue

Use the repository issue tracker:

- [Open GitHub issues](https://github.com/greenmice-technologies/hellomnemo.com/issues)

When possible, include:

- what you expected
- what actually happened
- platform and app context
- screenshots or sample data when safe to share
