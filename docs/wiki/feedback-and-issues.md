## Use issues for

- bug reports
- documentation corrections
- product feedback
- feature requests
- onboarding friction reports

## Before opening an issue

- check the [FAQ]({{ site.baseurl }}/docs/wiki/faq/)
- check [Project status]({{ site.baseurl }}/docs/wiki/project-status/)
- check [Troubleshooting]({{ site.baseurl }}/docs/wiki/troubleshooting/)

## Open an issue

Use the repository issue tracker:

- [Open GitHub issues](https://github.com/greenmice-technologies/hellomnemo.com/issues)

When possible, include:

- what you expected
- what actually happened
- platform and app context
- screenshots or sample data when safe to share
